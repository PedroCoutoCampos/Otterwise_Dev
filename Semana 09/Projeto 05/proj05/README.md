Roteiro de projeto: 
Proposta A proposta deste projeto é praticar a componentização de layouts. A partir disso, os estudantes deverão replicar um quadro utilizado para o acompanhamento de andamento de tarefas, conhecido como Kanban. Essa técnica visa controlar o fluxo de produção seja de um projeto ou tarefas de modo geral. Ao buscar mais conhecimento sobre métodos ágeiso Kanban certamente será um dos métodos que aparecerão, pois é com um entre utilizado em diversas empresas. Geralmente os quadros do Kanban tem as colunas:

●TODO (a fazer): É onde ficam as tarefas que precisam ser feitas;

●INPROGRESS(em progresso):Tarefas que já foram separadas por algum membro do time e estão sendo feitas;

●REVIEW(revisão):Tarefas que já foram finalizadas e precisam da revisão final para serem marcadas como prontas;

●DONE (pronto): Tarefas que já foram revisadas e foram marcadas como prontas. O layout que deve ser recriado é do quadro Kanban abaixo:

●Pixel perfect não será cobrado, mas tente implementar o mais perto possível do layout;

●Não é necessário pensar em layout mobile;

●A componentização será avaliada, então não faça tudo em um componente só;

●Não será cobrado a parte de funcionamento do quadro kanban, apenas o layout.Entrega do projeto

●Cada grupo pode ter no máximo 2 pessoas;

●Deve ser criado um novo repositório onde os membros do grupo irão trabalhar;

●O grupo deve entrar em acordo e escolher quem fará cada componente;

●Serão avaliados commits, PR’s e trabalho em equipe;

●A entrega final deve ser o link do repositório e cada membro do time deve enviar via formulário
